React-WebKTL Instruction Manual

Lasted Edited: July 24, 2024

The React-WebKTL is a web based software to create engineering interfaces for developing Keck and Lick instruments. It includes functionality for monitoring and controlling the instruments during the engineering process. It uses KTL services keyword and value pairs over a WebSocket connection to provide an interface with data and updates in real-time. With this software, engineers and other people developing systems will be able to create simple GUIs for KTL services.

General Information

What are WebSockets?

WebSockets are a communication protocol that allows for almost continuous communication after an initial handshake, which contains all the identification information required for data exchange. WebSockets are a good choice when two way communication is needed between client and server. Two way communication is possible due to the active connection after a handshake is established and the server can send data to the client without having first received a request. The connection remains open until either side closes the connection.

What is React?

React is a JavaScript library that aims to streamline the tedious and complex process of building interactive user interfaces. With React, developers can create a collection of reusable components, each responsible for rendering a small portion of HTML code. These components can be combined and nested to build intricate web pages. React’s component-based framework not only enhances code reusability and maintainability, but also improves the overall performance of applications by updating and rendering only the components that need to change, rather than the entire page.

Why WebSockets and React?

WebSockets and React together provide unmatched benefits in enhancing and simplifying  the development of a KTL services interface. WebSockets enable two-way communication, allowing the application to receive updates instantly without requiring the user to send repeated requests. This ensures that any changes in data are promptly reflected. In combination with React’s dynamic rendering capabilities, these real-time updates can be seamlessly displayed to the user as soon as they arrive. Using WebSockets and React not only streamlines the development process, but also creates a more responsive and engaging user experience.

Check Out the Possibilities

Server Side

To make the WebSocket protocol handshake the user must know the url of their service and send their request to a ws:// or wss:// url. This will be the url to which the WebSocket server will respond. Once the connection is established, the user can filter the data received by sending it a request with the data of the simple command line requesting data from the specific service.

Client Side

Display Keyword Value

Given any valid keyword, the corresponding value is displayed in real-time. Label can be adjusted as needed, or left off for no additional text displayed.

Keyword Button

Functional buttons can be created with a valid keyword. Label, color variants, and value submitted on click can be adjusted as needed. Clicking the button will send a modify request for keyword and update keyword value in real-time.

Keyword Dropdown Button

Functional dropdown button on keyword with adjustable label, and options. Selecting a new option from dropdown will send a modify request for keyword and update keyword value in real-time. If wish to populate options based on value from additional keyword, include additional keyword.

Keyword Form

Functional input forms can be created with a valid keyword. Label for input can be adjusted as needed. Input will send a modify request for keyword and update keyword value in real-time.

 

Keyword Boolean Checkbox

Functional true or false checkbox can be created with a valid keyword. Label for checkbox can be adjusted as needed. Input will send a modify request for keyword and update keyword boolean value in real-time. Functional for ‘true’, ‘yes’, ‘on’, and 1.

Keyword Checkboxes

Functional checkboxes can be created with a valid keyword. Labels for checkboxes can be adjusted as needed. By checking a box, a modify request will be sent for the keyword and update the value in real-time.

 

Keyword Boolean Switch

Functional switches can be created with a valid keyword. Label for switch can be adjusted as needed. By changing the switch, a modify request will be sent for the keyword and update the boolean value in real-time. Functional for ‘true’, ‘yes’, ‘on’, and 1.

Keyword Value Table

Given a list of keyword strings, this component will build a simple time of keywords and values. List can be as long or as short as the user wishes. Table can also include additional data from metadata tag.

Keyword History Plot

Given a WebSocket URL, service name, and keywords, this component generates a line graph displaying the values of the specified keywords over time using open source library Plotly. Users can provide an array of strings to set custom time options (from, to, and interval). If no time options are specified, default options will be used. Additionally, by toggling the switch on the right, users can replace the dropdown boxes with an input box, allowing them to freely enter their desired time range.

Data Container with Two Columns

This component enables the user to create nicely formatted web pages using React Bootstrap’s grid system. By passing in contents which might contain the essential components, user’s can mix and match to create a versatile site. The possibilities are endless with this component because you can continuously nest components within components until the desired look is reached.

Data Container Carousel

This component enables users to create a carousel data container where they can display more features and data. The carousel will automate on its own if left untouched, but users can toggle between pages in the carousel by clicking on the far left or far right of its current contents.

Sidebar

Functional Sidebar to toggle between pages. Has an expanded and collapsed state for an optimal user experience.

Understanding File Structure

Before jumping in creating your own pages, it is important to understand the folder structure of React-WebKTL inside the src folder. The src folder contains three main folders, components, context, and pages.

components

The `components` folder holds all the different GUI components needed for a functional UCO engineering interface. The important file to note in the `components` folder that may need attention is, `SideBar.js`. `SideBar.js` is what creates the sidebar navigation menu on the web page and as more web pages are created, you will need to add more `NavLink` options.

context

The `context` folder hold the files, `createWebSocketContext.js` and `WebSocketProviders.js`. These files work simultaneously together to provide proper functionality to our web pages via the WebSocket connection they are able to provide.

pages

The `pages` folder holds all the different files for each page in your application. As the interface needs increase, expect this folder to grow in file size to accommodate greater complexity in your React-WebKTL web page. `Home.js` should act as the dashboard which implements the other pages as seen in the demonstration. Feel free to change, add, and delete pages as needed for your application.

App.js

Located in the `src` folder, `App.js` is a core component of React-WebKTL. Taking a look at the file, the `WebSocketProvider` wraps the entire application, providing WebSocket functionality through the `url` prop. `BrowserRouter` wraps the routing logic, enabling client-side routing.

Inside the `BrowserRouter,` the `SideBar` component is rendered with a prop `instrumentName`, which can be edited to the user’s preference. Sidebar provides navigation throughout the application. Within `SideBar`, the `Routes` component defines the various routes for your webpage. It ensures that different components (`Home` page and `ExamplePage`) are rendered based on the URL path. Without `App.js`, the file would cease to run.

Getting Started

Creating Your Own WebKTL-React App

  1. Copy the Repository Link

The link can be found at https://github.com/ashnakashima/Webktl-React and can be easily copied by clicking on the “Clone” or “Code” button and selecting the appropriate option to copy the link.

  1. Open VS Code or other IDE Preference and Navigate to Your Desired Folder

If you wish to use an IDE, open it and navigate to the folder where you want to store the repository on your local machine.

  1. Clone the Repository

Once in the desired folder, open the integrated terminal. In the terminal, type the following command:

git clone https://github.com/ashnakashima/Webktl-React.git

  1. Navigate to the Clone Repository

After cloning is complete, navigate into the cloned repository by typing the following command in the terminal:

cd Webktl-React

  1. Install Dependencies

To ensure that the project runs smoothly, you need to install these dependencies. Type the following command in the terminal to install the necessary dependencies:

npm install

  1. Start the Project

Once all the dependencies are installed, you are ready to start the project. Type the command in the terminal:

npm start

WebSocket Handshake

To make WebSocket handshake with the proper url related to your instrument, there are a few adjustments that must be made in `App.js`. Towards the top of the file you will see a box declaring the strings, url, serviceName, and instrumentName. Adjust the `url` string variable to reflect the WebSocket connection to your server. The string variable `serviceName` should be changed to the name of the service you wish to create the interface for. Keep in mind, this should be the service name for all keyword values that will be displayed in the web page. Finally, adjust `instrumentName` to reflect what you are working on. This test will be displayed in the upper left corner of your side navigation bar when toggled opened to enable more customization by the user.

SideBar

As your page grows, you may want to add new tabs to the side navigation bar to organize the data and functions. To add your new tab, ensure that under the pages folder, you have created a new .js file to represent that tab. Then going into SideBar.js in the components folder, add the the following node under the last NavLink tag.

<NavLink to="/url-extension-to-direct-page" className={({isActive}) => (isActive ? 'activeClicked' : '')}>
  <CDBSidebarMenuItem icon={“icon-of-your-choice”}>
      {Text Displayed for the Tab}
   </CBDSidebarMenuItem?
</NavLink>

Anywhere the text is red, it can be replaced with text of your choosing. The “url-extension-to-direct-page” will be the url to direct the page on click. ‘Icon-of-your-choice’ will display an icon at the expanded and collapsed state of your sidebar. Experiment with which icon looks best and fits your page theme. The ‘Text Displayed for the Tab’ will display the tab title, make it clear and concise.

To make the navbar functional, you will need to go to ‘App.js’. Under the previous routes, add a new route:

<Route path="/url-extension-to-direct-page" element={<NameOfJSFile/>} />

Ensure the url extension is consistent between SideBar.js and App.js. ‘NameOfJSFile’ is the name of the .js file you created which holds all the GUI components of this new tab.

Finally, at the top of App.js, add the following import statement.

import NameOfJSFile from"./pages/nameOfFile";
